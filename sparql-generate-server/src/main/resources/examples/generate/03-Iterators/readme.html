In this setup we iteratively generate RDF from different air quality JSON reports.

The syntax of the ITERATOR clause is the following: 

ITERATOR <iterator>(arg_1, arg_2, ... arg_n) AS ?var_1 ?var_2 ... ?var_m

Where <iterator> is the IRI of the SPARQL-Generate iterator function, which is similar to a SPARQL 1.1 Function, except it returns a list of RDF Terms instead of just one. The evaluation of the iterator function over the arguments args generates bindings for one or more variables, depending on the iterator function.

Remember: iterator functions are documented at their URI (e.g., iter:for). The set of available binding functions is available at http://w3id.org/sparql-generate/iter/

SPARQL-Generate Iterator functions are defined to generate RDF from JSON, XML, HTML, CSV, and plain text.

  • Custom SPARQL binding functions all take a set of RDF terms as input, and output zero or one RDF term. They all have namespace http://w3id.org/sparql-generate/fn/ with preferred prefix fun.
  • Iterator functions are used in the ITERATOR clause. They all take a set of RDF terms as input, and output zero or more RDF terms. They all have namespace http://w3id.org/sparql-generate/iter/ with preferred prefix iter.
